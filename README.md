# Fuzzy DB Quality Assessment

Система нечёткой оценки качества баз данных на основе метрик структуры.  
Позволяет анализировать схему PostgreSQL и вычислять интегральный показатель «качество БД» с помощью аппарата нечёткой логики.

## 📌 Возможности

- **Подключение к PostgreSQL** и автоматический сбор структурных метрик.
- **Расчёт пяти ключевых показателей** схемы данных:
  - доля составных первичных ключей,
  - доля нецелочисленных первичных ключей,
  - доля CHAR‑атрибутов с неоптимальным расходом памяти,
  - доля внешних ключей без `ON UPDATE`,
  - доля внешних ключей без `ON DELETE`.
- **Нечёткая система оценки** с треугольными функциями принадлежности.
- **Дефаззификация методом центра тяжести** для итоговой оценки.
- **Визуализация** функций принадлежности, полигона решения и матриц правил.
- **Интерактивный веб‑интерфейс** на Streamlit.
- **Генерация новых правил** на основе композиции матриц отношений.

## 🧠 Как работает нечёткая система

1. **Фаззификация** – каждая входная метрика (число от 0 до 1) преобразуется в степени принадлежности к термам `мало`, `средне`, `много`.
2. **Агрегация правил** – для каждого правила вычисляется минимум степеней принадлежности условий (логическое «И»).
3. **Активация** – выходные термы качества (`очень_плохое`, `плохое`, `нормальное`, `хорошее`, `очень_хорошее`) усекаются по уровню активации.
4. **Аккумуляция** – усечённые функции принадлежности объединяются максимумом.
5. **Дефаззификация** – вычисляется центр тяжести получившейся фигуры; его координата `x` даёт итоговую оценку качества.

## 🚀 Быстрый старт

### 1. Локальный запуск (без GUI)

Для теста без Streamlit используйте `main.py`. Убедитесь, что в `utils/rules.json` прописаны правила.

```bash
python main.py
```

Скрипт:
- загрузит правила,
- выведет матрицы отношений,
- сгенерирует новые правила через композицию,
- выполнит полный цикл нечёткой оценки на тестовых входных данных,
- покажет графики функций принадлежности и дефаззификации.

### 2. Запуск веб‑интерфейса (Streamlit)

```bash
streamlit run app.py
```

Откройте браузер по адресу `http://localhost:8501`.

**Инструкция в интерфейсе:**
1. В левой панели введите параметры подключения к PostgreSQL.
2. Нажмите **«Подключиться»**.
3. После успешного соединения нажмите **«Оценить качество БД»**.
4. Через несколько секунд отобразятся метрики и графики (выводятся в консоль, где запущен Streamlit).

## 📊 Сбор метрик БД

Класс `UserDBManager` подключается к указанной схеме PostgreSQL и вычисляет:

| Метрика | Описание |
|--------|----------|
| `доля_составных_ключей` | Доля таблиц с первичным ключом, состоящим более чем из одного столбца. |
| `доля_не_целочисленных_ключей` | Доля таблиц, где хотя бы один столбец первичного ключа имеет нецелочисленный тип. |
| `доля_char_атрибутов_с_растратой_памяти` | Доля CHAR‑столбцов (не VARCHAR), у которых длина > 1. |
| `доля_таблиц_без_on_update` | Доля внешних ключей с правилом `ON UPDATE NO ACTION` или `RESTRICT`. |
| `доля_таблиц_без_on_delete` | Доля внешних ключей с правилом `ON DELETE NO ACTION` или `RESTRICT`. |

Все значения лежат в диапазоне `[0, 1]`.

## 🧾 База правил (`utils/rules.json`)

Правила записываются в формате:
```json
{
  "db_quality_rules": [
    {
      "rule": "ЕСЛИ доля_составных_ключей – мало И доля_не_целочисленных_ключей – мало ТО качество – очень_хорошее",
      "weight": 1.0
    },
    ...
  ]
}
```

Поддерживаются:
- одиночные условия: `ЕСЛИ параметр – терм ТО качество – терм`;
- составные условия с оператором `И`: `ЕСЛИ параметр1 – терм1 И параметр2 – терм2 ТО качество – терм`.

Вес правила (weight) участвует в построении матриц отношений.

## 📈 Визуализация

При запуске через `main.py` выводятся:
- тепловые карты матриц отношений (параметры × термы качества);
- графики всех функций принадлежности качества;
- полигон решения с отмеченными вершинами и центром тяжести.

Все графики строятся с помощью `matplotlib` и `seaborn`.

## 🔧 Дополнительные возможности

- **Генерация новых правил** – метод `generate_rules_from_composition` в `RuleParser` создаёт правила на основе композиции матриц двух параметров.
- **Дополнение матриц** – вычисление complement (1 – value) для анализа «отрицательных» закономерностей.
- **Транспонирование матриц** – позволяет строить обратные отношения «качество → параметры».
